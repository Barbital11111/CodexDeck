## 更新日志
- v2.1.3
  1. 为 Grok 4.5 / 4.5 latest 增加 `Low / Medium / High` 推理档位。
  2. 修复多模型模式大图片或长上下文请求超过 2 MiB 时触发 413。
- v2.1.2
  1. 支持 Codex `26.707.72221`。
- v2.1.1
  1. 优化多模型选择器的尺寸与动效，统一当前值和菜单中的六档英文推理强度，并兼容两版 Codex `app.asar`。
  2. API 配置改为本地快速保存，模型能力与额度保留手动探测入口。
  3. 增加受控 Codex 快速校验与旧副本、patch/provider 备份清理，减少重复启动等待和磁盘占用。
  4. 修复受控副本已完成 patch 但未成功晋级，启动后回退官方界面的问题。
  5. 修复第三方模型错误继承 GPT 六档推理强度的问题，按 MiniMax-M3、GLM-5.2 和 DeepSeek 的实际能力生成档位。
  ![CodexDeck 2.1.1 多模型选择器](docs/assets/model-picker-v2.1.1.png)
- v2.1.0
  1. 清理前端构建产物，避免历史 Hash 静态资源被打入安装包。
  2. 更换自动更新签名密钥，后续版本将使用新密钥验证更新。
- v2.0.9
  1. 新增“禁用 Codex GPU 加速”设置，可缓解部分环境下 Codex Desktop 闪退、打字卡顿和卡死问题。
  2. 新增并完善 Codex 多模型模式，支持受控 Codex 副本和新版 Codex `app.asar` 多模型 patch。
  3. 优化多模型模式启动速度和版本同步逻辑，减少重复 patch 检查，并修复多模型模式关闭后仍被恢复的问题。
  4. 补齐 gpt-5.5 基础指令资源与相关 profile 写入支持。
- v2.0.6
  1. 新增多供应商 API 接入体验：添加账号页提供 MiniMax、Xiaomi MiMo、Xiaomi MiMo Token Plan、DeepSeek、Z.AI GLM、Kimi 和自定义供应商入口。
  2. 新增模型目录管理：支持探测上游 `/models`，可分别维护菜单模型 ID、显示名称、实际请求模型、启用状态和上下文窗口。
  3. 新增本地模型路由模式：可聚合多个 API 账号中启用的模型，按模型名转发到对应上游，并在启动时写入 Codex 配置。
  4. API 额度显示补强：DeepSeek、MiniMax、GLM、Kimi 等可查询平台支持刷新额度；已知订阅平台支持手动订阅标签，MiMo Token Plan 暂不查询余额。
  5. 上下文窗口显示改为 `K/M` 格式；明确模型保留已知窗口，未知非 GPT 模型默认推荐 `256K`，GPT 系列不自动补写。
  6. classic / modern 两套 PC 端 UI 皮肤改为并行开发，classic 恢复原版布局与蓝白观感，modern 保留新版暖橙方向。
  7. 修复添加 API 账号和编辑 API 账号时的多处 UI 问题，包括预设供应商状态、订阅标签显示、侧栏滚动、开关尺寸、按钮尺寸和模型上下文输入展示。
  8. 移除旧混合 relay proxy 模块，混合模式和路由模式的请求链路收口到当前 provider/profile 写入与本地模型路由实现。
- v2.0.5
  1. CodexDeck 主线整理为账号、API、混合模式、额度显示、通知数据源和增强启动的稳定版本。
  2. 修复混合模式请求链路：本地 relay 会剥离普通文本请求中默认携带的 `image_generation` 工具，避免不支持生图的上游被误判失败；真实生图分流交由外部中转服务处理。
  3. 修复 API 导入页余额显示：新增“额度来源”选择，并在创建 API 条目时正确保存平台账号、密码和订阅模式。
  4. 修复 OAuth 授权链接生成：默认回调端口被系统拒绝或保留时，会自动回退到本地空闲端口。
  5. 修复公开仓库、Issue、Release 和自动更新地址，统一指向 `Barbital11111/CodexDeck`。
  6. 保留历史安装身份 `com.carry.codex-tools`，确保同版本覆盖安装和旧数据目录兼容。
  7. 远程控制运行时不再作为 CodexDeck 主仓库源码的一部分；需要远控能力时应作为外部工具单独部署。
